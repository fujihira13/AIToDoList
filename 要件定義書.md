# アイゼンハワーマトリクス TODO アプリ 要件定義（v1 ドラフト）

## 1. システム概要

- **名称（仮）：** Eisenhower Board（社内タスクボード）
- **目的：**
  - 職員のタスクをアイゼンハワーマトリクス（4 象限）で一覧・共有し、「誰が」「どのタスクを」「どれくらい急ぎ・重要か」を一目で把握できるようにする。
  - 遊び心のある見た目（表情・写真）で、堅すぎないタスク管理ツールとする。
- **導入形態：**
  - 既存ツールを置き換えるのではなく、新しく試験導入する社内ツールとしてスタート。

---

## 2. 利用対象・規模

- **利用者：**
  - 職員 約 10 名程度（初期想定）
  - 全員がタスクの閲覧・登録・編集を行える。
- **権限：**
  - v1 では細かい権限分けは行わず、「全員が全タスクを編集可能」とする。
  - 将来的な拡張（担当者のみ編集可 / 管理者ロールなど）は別途検討。

---

## 3. 利用環境

- **実行環境：**
  - まずはローカル開発環境で動作させることを前提とする。
  - デプロイ（社内サーバ / クラウド）は v2 以降で検討。
- **クライアント環境：**
  - PC ブラウザからの利用を想定。
  - 想定ブラウザ：Google Chrome 最新版（※他ブラウザ対応は優先度低）。
- **レスポンシブ対応：**
  - スマホ対応は必須要件としない（あったら嬉しいレベル）。

---

## 4. 機能要件

### 4.1 アイゼンハワーマトリクスボード

- **4 象限の定義：**
  1. 重要かつ緊急（左上）
  2. 重要だが緊急ではない（右上）
  3. 緊急だが重要ではない（左下）
  4. 重要でも緊急でもない（右下）
- **ボード構成：**
  - 画面上に 4 つの象限をグリッド表示する。
  - 全職員共通のボードを 1 枚だけ表示する。
- **タスクカード表示項目：**
  - タイトル
  - 担当者の名前
  - 担当者のプロフィール写真（1 人につき 1 枚）
  - ステータス（未着手 / 進行中 / 完了）
  - 期限（締切日・任意）
  - 優先度（高 / 中 / 低）
- **ドラッグ＆ドロップ：**
  - タスクカードをマウスでドラッグし、別の象限へドロップすることで「象限」が変更される。
  - ドロップ時に、バックエンドへ非同期で象限変更を反映する（Ajax / Fetch 等）。
- **見た目の遊び心（表情など）：**
  - 各象限ごとに異なる「表情アイコン」や背景イメージを表示する。
    - 例：
      - 左上：怒っている／焦っているイメージ
      - 右下：穏やかな表情・リラックスしたイメージ
  - v1 では、共通のアイコン or イラストで表現（個人ごとの表情差し替えは将来拡張）。

---

### 4.2 タスク管理機能

- **タスク作成：**
  - ボード画面から「タスク追加」ボタンで新規作成フォームを表示する。
  - 入力項目：
    - タイトル（必須）
    - 詳細説明（任意）
    - 担当者（必須 / 職員から 1 人選択）
    - 登録者（作成者）※v1 では自動設定または任意入力でも可
    - 期限（締切日・任意）
    - ステータス（未着手 / 進行中 / 完了）
    - 所属部署（選択またはテキスト）
    - 優先度（高 / 中 / 低）
    - 象限（1〜4）※初期値はフォームで選択
- **タスク編集：**
  - タスクカードをクリックすると編集フォームを表示する。
  - 上記項目の編集が可能。
  - ドラッグ＆ドロップによる象限変更も編集扱いとして保存する。
- **タスク削除：**
  - v1 では、全員がタスク削除可能とする。
- **ステータス管理：**
  - ステータス変更により視覚的な違い（色味・ラベルなど）をつける。
  - 完了タスクの扱い：
    - 初期案としてはボード上に残しつつ、薄く表示する。

---

### 4.3 職員（ユーザー）管理機能

- **職員情報：**
  - 1 人につき以下の情報を保持する：
    - 氏名
    - 所属部署
    - プロフィール写真（1 枚）
- **写真：**
  - ローカル開発環境では、`MEDIA`ディレクトリへのアップロードで対応する。
  - 画像はタスクカード内で小さく（丸型アイコンなど）表示する。
- **登録・編集方法：**
  - v1 では Django 管理画面（admin）を利用し、管理者が職員情報を登録・編集する想定。
  - 将来的に、一般画面から職員一覧・編集画面を実装することも可能。

---

### 4.4 ログイン・認証

- **v1：**
  - ログイン機能は実装しない。
  - ローカル環境での利用を前提とし、アクセス制御は行わない。
- **将来拡張案（v2 以降）：**
  - Django 標準のユーザーモデルを利用した ID/パスワード認証。
  - 職員ごとにアカウントを紐づけ、担当者・登録者と連動。
  - 権限ロール（管理者 / 一般職員）追加。

---

## 5. 非機能要件（v1 想定）

- **パフォーマンス：**
  - 同時利用者 10 名程度を想定。
  - ローカル環境でストレスなく動作する程度を目標とする。
- **信頼性 / バックアップ：**
  - v1 では開発・試験用のため、本格的なバックアップ要件は規定しない。
  - 重要データ化した場合、DB バックアップ方針を別途策定する。
- **拡張性：**
  - Django 標準の構造（モデル / ビュー / テンプレート）に従い、画面・機能追加がしやすい構成とする。

---

## 6. 画面一覧（v1）

1. **マトリクスボード画面**
   - 4 象限タスク一覧表示
   - タスクのドラッグ＆ドロップ移動
   - タスク新規作成ボタン
   - タスク編集（クリックでフォーム表示）
2. **タスク作成・編集フォーム**
   - モーダルまたは別ページで表示
   - タスク項目一式の入力・更新
3. **職員管理（Django 管理画面）**
   - 管理者が職員の名前・部署・写真を登録・編集

---

## 7. データ項目定義（ドラフト）

### 7.1 職員（Staff）

| 項目名     | 型       | 必須 | 説明                          |
| ---------- | -------- | ---- | ----------------------------- |
| id         | 整数     | ○    | 主キー                        |
| name       | 文字列   | ○    | 氏名                          |
| department | 文字列   | △    | 所属部署（空でも可）          |
| photo      | 画像パス | △    | プロフィール写真（1 人 1 枚） |
| created_at | 日時     | ○    | 登録日時                      |
| updated_at | 日時     | ○    | 更新日時                      |

### 7.2 タスク（Task）

| 項目名      | 型           | 必須 | 説明                                      |
| ----------- | ------------ | ---- | ----------------------------------------- |
| id          | 整数         | ○    | 主キー                                    |
| title       | 文字列       | ○    | タイトル                                  |
| description | テキスト     | △    | 詳細説明                                  |
| owner       | 外部キー     | ○    | 担当者（Staff への FK、1 人のみ）         |
| created_by  | 文字列 or FK | △    | 登録者（v1 では任意／将来 User 紐づけ可） |
| due_date    | 日付         | △    | 締切日                                    |
| status      | 列挙         | ○    | 未着手 / 進行中 / 完了                    |
| department  | 文字列       | △    | 所属部署（owner と同じでも別でも OK）     |
| priority    | 列挙         | ○    | 高 / 中 / 低                              |
| quadrant    | 整数         | ○    | 1〜4（アイゼンハワーマトリクスの象限）    |
| created_at  | 日時         | ○    | 登録日時                                  |
| updated_at  | 日時         | ○    | 更新日時                                  |

---

## 8. v1 スコープ（まず作る範囲）

**v1 で必ず実装するもの：**

- 全職員共通の 4 象限ボード画面
- タスクの新規登録・編集・削除
- タスクのドラッグ＆ドロップによる象限変更
- 職員の登録（Django admin）＋プロフィール写真 1 枚
- タスクカードに担当者の顔写真・ステータス・期限などを表示
- 象限ごとに異なる「表情アイコンまたは背景」を表示（簡易な遊び要素）

**v1 では後回しにするもの：**

- ログイン機能・権限分け
- メール通知・チャット通知
- 部署ごとの別ボード表示
- スマホ最適化
- 詳細な変更履歴機能
